de meeste code is lelijk. Maar het hoeft niet. De meeste programmeurs maken code lelijk. Hetgeen er het meeste aan scheelt is altijd naamgeving. Afkortingen zegevieren, C/C++ programmeurs vinden het logisch twee verschillende dingen dezelfde namen te geven met hoofdlettergebruik als verschil. De tweede ramp is interface indeling die niet deugt. Een andere ramp is gebrek aan verschillende abstractie niveaus. Een andere ramp is indeling van code modules. Robuustheid en corectheid ontbreekt ook vaak. Meestal ontbreken al deze facetten van het goed programmeren. Ik vraag me altijd af waarom het nou zo moeilijk is en dat zou moeten betekenen dat het gemakkelijk valt uit te leggen. Dus schrijf ik dit boek om samen te vatten wat de meeste programmeurs fout doen door te zeggen wat goed is aan goed programmeerwerk..
